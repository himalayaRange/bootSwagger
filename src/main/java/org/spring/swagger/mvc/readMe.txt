访问权限设计原理：
1.在客户端构造http get post 请求，将数据加密后放入body中，并在http参数中放入调用方的签名

2.服务端接收到请求，提取签名进行校验，校验通过后从request body中提取密文解密，然后进行后续处理并返回请求结果

说明：
   对于RSA算法的签名中公钥和私钥的生成工具在GenerateUtil.createRSAKey(),后续可以通过盐值等方式将其转化成商户号和秘钥，
 Sign中针对RSA算法中通过商户号和秘钥获取RSA中提供的公钥和私钥
  相应的CheckSign针对RSA算法的验证也需要进行修改，即商户号和秘钥获取RSA中提供的公钥和私钥，否则解析错误
